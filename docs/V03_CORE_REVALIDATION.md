# TRI RELAY v0.3 — コア再検証版 決定記録

2026-07-15。これは実在人物本人の発言ではなく、公開作品・講演・開発思想から
判断軸を再構成した架空の役員会を基にした制作記録である。固有の画風や表現を
模倣するための資料ではない。

## なぜv0.2を本番候補から降格したか

v0.2は中央リレー、採掘→製造→防衛、決定論エンジン、スマホ縦画面という
企画の核を保っていた。一方、「固定手順対策に成功した」という判定は、
`採掘→採掘→製造→防衛`の1周期だけを比較した不十分な検証だった。
同じ条件で別の5手周期を試すと、判断型botを上回る勝率が出たため、
v0.2文書の成功判定を撤回した。

また、背景に固定の敵軍が描かれ、実際の敵数と画面が一致していなかった。
設備SVG、配線、都市状態もスマホCSSで隠れており、「送電で世界を動かす」
ゲームの因果を小さなラベルだけで伝えていた。

## v0.3で採用した修正

1. 各回路へ0〜100の可視負荷を追加。次の送電後の負荷、状態、出力を操作前に表示する。
2. 周波数一致は回路を冷却し、不一致は負荷を上げる。未使用回路もパルス単位で冷える。
3. 鉱石・弾薬・接敵が明示条件を満たす緊急送電は、表示上「緊急需要・負荷軽減」とし、
   周波数を外しても負荷上昇を20、出力を125%にする。隠れた救済にはしない。
4. 共鳴は連続判断として扱い、通常不一致では1段階減衰する。空設備・妨害では消失し、
   表示済みの緊急送電だけ保持する。超出力は対象回路を冷却する。
5. 2〜8手の全固定周期を自動探索し、強い周期を実ランで再試験する回帰ゲートを追加する。
6. 初回の固定ガイドは採掘→製造→防衛を2巡する6送電までとし、第1ウェーブ全体を
   固定手順の練習にしない。7送電目からseeded forecastへ移る。
7. スマホ画面へ採掘機、製造炉、砲台、3本の配線、送電粒子を復帰させる。
8. 負荷、重要警告、実働敵の名称と輪郭を12px級の判断情報として表示する。
9. 都市の明滅と侵攻圧を、耐久、敵数、全敵中の最大進行度から縦横とも動的に描画する。
10. 縦横のv03背景から固定敵を除き、空の進行路と非定形の煙・警戒光だけを残す。
11. ChatGPT埋め込みと一般iframeを分離し、右側ホストUIの安全余白を必要な環境だけへ適用する。
12. v0.2途中ランは新ルールと混在させず破棄し、通算記録・設定だけを維持する。

## 維持した範囲

- 1タップで120度回転する中央リレー
- 採掘→製造→防衛の資源連鎖
- 決定論的な固定タイムステップ
- 6ウェーブ、通常敵3種＋ボス、強化12種
- 都市耐久4。耐久6の比較は簡易色優先を過度に救うため不採用
- 初回だけの3手ガイド、途中保存、復帰時ポーズ
- Vercel／Sites両ビルド、CrazyGames環境限定SDK、広告無効

v0.3はランを長くする版ではない。まず「操作ごとの因果」と「固定作業ではない判断」を
再検証するため、敵、ウェーブ、強化、プロトコルの追加制作を凍結している。

## 自動検証の扱い

`npm run simulate`は、純粋な色追従、色優先＋出力不能時だけ代替、判断型の3政策、
全固定周期の簡易出力監査、上位周期の6ウェーブ実ラン対戦を同時に行う。

最終候補の拡張監査では、同一1,000 seedで判断型79.8%、純色追従29.9%、
色優先＋代替67.2%だった。判断型は後者を12.6ポイント、平均スコアを3,239上回った。
耐久6の比較では差が5.4ポイントまで縮んだため、耐久4を採用した。

2〜8手の9,837固定周期では、最良固定出力79.184%に対し適応基準100%だった。
上位8周期を各80実ランへ通すと最良固定1勝、同seedの判断型57勝だった。
初回ウェーブは100 seedで20.175〜24.575秒に収まった。

これらは既知の自動攻略を見落とさないための回帰証拠であり、人間が面白いと感じる
証明ではない。特に「色優先＋出力不能時だけ代替」が人間にも単調に感じられないかは、
新規テストで最優先観測する。
