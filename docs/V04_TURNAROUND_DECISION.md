# TRI RELAY v0.4 立て直し役員会決定

更新日: 2026-07-19
対象: `v0.4.0` コア再検証版
公開判定: クローズド検証のみ。Basic Launch候補ではない。

> この文書は、各氏本人の発言・承認・見解ではありません。公開されている仕事から抽出した異なる判断軸を用いた、架空の社内レビューです。

## 監査で確定した事実

- 中央リレーを120度回し、採掘→製造→防衛をつなぐ核は維持されている。
- `v0.3`で検証した固定手順は一種類だけで、別の5手周期が判断型botを上回った。したがって「固定手順対策済み」という以前の評価を撤回する。
- 6ウェーブの適応型bot平均勝利時間は約4分24秒で、企画目標の9分30秒〜11分には届かない。
- ただし、初見理解を証明せずに12ウェーブへ延長すると、面白さではなく待ち時間を増やす危険がある。
- W1から周波数・共鳴・回路負荷・強化を同時提示しており、最初の90秒に判断規則を詰め込みすぎていた。
- 計測コードは存在しても、新規人間テストの実測値は存在しない。自動テストと人間KPIは別の証拠である。
- 技術基盤、決定論、復帰、軽量アセット、両ビルドは再利用価値が高く、作り直し対象ではない。

## 第1巡: 各役員の判断

### 宮本茂 — 一手の因果

最初に守るべきものは「一回押すと、どこへ電気が流れ、何が増え、何を防いだか」が読めること。W1では採掘→製造→防衛だけを体で覚えさせ、周波数・共鳴・負荷は後から一つずつ導入する。画面が答えを推薦し続ける設計は、正解ボタンを押す作業になるため廃止する。

### 小島秀夫 — 操作が世界を変えること

最後の都市という舞台は残す。ただし物語を増やすのではなく、送電、機械の稼働、砲撃、都市耐久がプレイヤー操作へ反応することを優先する。勝敗後も「なぜそう見えたか」を、記録した事実だけで振り返れるようにする。

### 宮崎英高 — 定石と代償

固定周期が高勝率なのは致命的だが、見えない罰で潰してはいけない。共鳴を外した時の漏電、空回し時の発熱、回路負荷を事前に表示し、プレイヤーが定石を崩す理由を理解できるようにする。ビルド差は恒久攻撃力ではなく、資源連鎖と戦闘の異なる経路から作る。

### ジョン・カーマック — 検証可能な規則

固定周期検証を一例から全探索へ変える。回転位相を正規化した長さ1〜8の原始周期1,318種類を走査し、強い候補を同じ実エンジンで再試験する。UI表示倍率と内部倍率、保存スキーマとエンジン入力、SDKの状態遷移を一つの定義へ揃える。

### 岩田聡 — 数字の意味

平均時間、初撃破、90秒到達、任意再挑戦を別々に記録する。チュートリアルを考えている時間も、画面を離れていない限り能動時間へ含める。自動テストを人間テストの代用にせず、新規10人を無言で観察し、合格するまでコンテンツ拡張を凍結する。

### 新川洋司 — 機械と機能の一致

浮いた説明カードを増やさず、設備、配線、砲台へ情報を結び付ける。W1で隠すのは機械そのものではなく、まだ使わない高度ルールである。色、発光、稼働方向を因果表示として使う。

### 天野喜孝 — 状態を語る都市

青白い生命、琥珀の採掘、紫の製造、赤い脅威という象徴は維持する。ただし固定背景がゲーム状態を偽ってはならない。静止画へ敵数や勝敗を焼き込まず、動的レイヤーだけが現在の危機を語る。

### 鳥山明 — 390pxでの即読性

重要語を省略記号で切らない。W1で必要なのは「採掘」「製造」「防衛」と押す場所だけ。小さい補助文字より大きな数値・輪郭・矢印を優先し、320×568と390×844で実機確認するまで合格扱いにしない。

## 相互反論

- 宮本→小島: 新しい物語演出より、一手の因果を先に完成させる。
- 小島→カーマック: 軽量化は世界を静止させる理由にならない。既存DOM/CSS/SVGの局所変化で十分に語れる。
- 宮崎→岩田: 既知の固定周期欠陥を残したまま人間テストをしても、欠陥の再確認になる。まず既知欠陥を直す。
- 岩田→宮崎: 欠陥修正後に難度・敵・ウェーブを増やしてはいけない。90秒理解ゲートが先である。
- カーマック→宮崎: 定石対策はすべて決定論的・可視・テスト可能でなければならない。隠し補正は採用しない。
- 新川→鳥山: 読みやすさのために設備をカードへ戻さず、設備本体の可動と短いラベルを大きくする。
- 鳥山→新川: 機械として美しくても小画面で機能が読めない部品は、商品画面では不採用にする。
- 天野→全員: 単なる計器盤に戻さず、都市の灯と危機色を状態へ連動させる。

## v0.4で実行する範囲

1. W1は基本連鎖だけ、W2で周波数と共鳴、W3で負荷を段階解禁する。
2. チュートリアル後の「次に押す正解」推薦を撤去し、事実と予報だけを表示する。
3. 固定周期を位相正規化して長さ1〜8まで探索し、CIで退行を止める。
4. 3ビルドの勝率下限と最大差を検証ゲートにする。
5. チュートリアル完了とActive Runを版付き・厳格検証で保存する。
6. CrazyGames SDKを現在の希望状態へ同期し、遅延初期化後の誤開始や二重通知を防ぐ。
7. 初撃破、能動90秒、再挑戦、操作密度、資源生産率を匿名・最小限で計測する。
8. 敗北後は因果を捏造せず、記録された上位2事実だけを表示する。
9. 新規10人の無言テスト手順とGO/NO-GO基準を固定する。

画像、動画、音声、敵数、ウェーブ数、課金、広告はこの版では増やさない。

## 検証記録

`v0.4`候補では次を検証対象とする。

- 自動テスト88件（型付き65件＋配備・表示・メディア・HTTP検証23件）
- lint
- ChatGPT Sites向けビルド
- Vercel/Next.js向けビルドと型検査
- 技術リリース容量・参照整合性
- 1,000シードの方策比較
- 320×568、390×844、横画面、iPhone Safariの実機確認
- 新規10人の無言プレイテスト

再構築後の正本値は、1,000シードで適応型89.4%、旧UI推薦81.2%、単純閾値2.7%、周波数追従19.1%、有効時周波数追従68.9%である。適応型の勝利平均は264.464秒（159.953パルス）。採掘主導76.9%、製造主導77.4%、防衛主導83.4%で、最低勝率76.9%、最大差6.5ポイントのゲートを通過した。長さ1〜8の原始・位相正規化固定周期1,318種類を簡易走査し、上位137候補を各80シードの実ランで再試験した結果、最良固定周期は17.5%、同じシードの適応型は85.0%で差67.5ポイントだった。これらは`SIMULATION_RUNS=1000 node --experimental-strip-types scripts/simulate.mts`で再現する値を正本とする。

作業領域復旧後の自動Chromium確認では、390×844で開始、W1段階表示、チュートリアル完了、ポーズ、最初の3分岐強化、W2表示、プライバシー画面を実操作した。320×568では開始操作、スクロール可能なプライバシー導線、W1の機械名・脅威・リレー・ポーズを確認し、実際の敗北結果画面が先頭から開いて見出しとラン記録を隠さないことも確認した。800×450と1366×768では開始画面を確認した。全対象で横はみ出し、通信失敗、コンソール例外、エラーオーバーレイは0件だった。ただし、これはヘッドレスChromiumのビューポート確認であり、ブラウザUIを含む実機、CrazyGames実iframe、iPhone Safariの合格記録ではない。

商用素材承認サブゲートは9件のアート／法務承認が`pending`のため未通過である。承認時は実在する承認者名・役割・日付も必須とする。さらに全体の公開ゲートは、プライバシー法務確認、実機確認、T01〜T10の人間テスト、プロダクトオーナー承認が未通過である。プライバシー文面は、運営者、連絡先、保持期間、利用する計測事業者へのリンクを法務担当者が確定するまで公開申請に使わない。存在しない情報を補完しない。

## 2026-07-19時点のCrazyGames要件との照合

- Basic Launchの主要ベンチマークは平均プレイ10分以上、D1保持10〜15%、1分到達80%以上、読込10秒未満、ビルド20MB未満。これは成功例の目安であり、審査合格を保証する固定採点表ではない。
- Basic LaunchのKPI取得にSDKは必須ではなく、統合しても広告は無効のままである。したがってv0.4で広告を追加しない。
- Full Implementationは新規ユーザーを即座にゲームへ入れることを求め、難しい場合も開始まで最大1クリックである。
- オンボーディングはゲーム内、コア機能だけ、スキップ可能、文字より視覚を優先することが推奨されている。v0.4の段階解禁と明示スキップはこの方向へ合わせる。
- DPR 1の各iframeとモバイルで読める必要があり、CrazyGamesが列挙する`800×450`モバイル表示も実機表へ加える。
