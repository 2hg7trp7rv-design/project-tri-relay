# TRI RELAY v0.4 新規プレイテスト手順

対象: ゲームを見たことがないT01〜T10
版: GitHubコミットSHAと配備URLを全記録へ固定する
目的: 最初の90秒で基本連鎖を理解し、本人の意思で次のランを始めるかを測る

## 禁止事項

- 観察者は、操作、色、周波数、共鳴、負荷、敵、強化を説明しない。
- 「そこを押して」「次は製造」などの誘導をしない。
- 失敗を防ぐために端末へ触らない。
- 再挑戦を促さない。
- 同僚や開発関係者を新規テスターに数えない。
- 不具合回を成功回として補正しない。原因と除外判断を別欄に残す。

安全・アクセシビリティ上の補助が必要な場合は実施してよいが、補助内容を記録し、標準群とは分けて解釈する。

## 事前準備

各テスターごとに新しいブラウザプロファイル、または本ゲームのLocal Storageを消した状態を用意する。次を記録する。

- テスターID: T01〜T10
- 実施日時
- コミットSHA、配備URL
- 端末、OS、ブラウザ、画面寸法
- 音声ON/OFF、入力方法
- 観察者
- 既知の類似ゲーム経験（終了後に聞く）

## 観察手順

### 0〜5秒

画面を見せ、説明せずに計時を始める。

記録:

- 最初に見た場所
- 5秒以内に押す対象を特定したか
- 最初の入力時刻
- 間違って操作した対象

### 開始〜最初の撃破

沈黙したまま次を時刻付きで記録する。

- 採掘への最初の送電
- 製造への最初の送電
- 防衛への最初の送電
- 最初の有効弾薬生成
- 最初の撃破
- チュートリアル完了
- 迷い、連打、説明文の長時間読解、誤タップ

自動イベント`first_kill`は撃破のゲーム内時刻を記録する。観察者は壁時計での時刻も別に記録する。

### 90秒

`active_90s_reached`は、ページが前面かつゲームがポーズされていない能動時間の累計90秒で発火する。ガイド中にプレイヤーが考えている時間も含む。タブ非表示、明示ポーズ、バックグラウンド時間は含めない。

観察者の壁時計90秒は別項目で記録し、能動時間と混同しない。

### ラン終了

勝敗に関係なく、結果画面を閉じる前に次を記録する。

- 能動時間、壁時計時間
- 最初の撃破までの時間
- 回転数、回転/秒
- 生産的パルス率
- 過負荷回数、オーバードライブ回数
- 最終ビルド
- 表示された事実ベースの振り返り

結果画面で沈黙を続け、本人が自発的に再挑戦した場合だけ「任意2ラン目」とする。観察者が質問または促しをした後の再挑戦は別集計にする。

## 終了後の質問

質問はこの順序で、一度だけ行う。選択肢を先に読まない。

1. 「このゲームで何をしていましたか？」
2. 「採掘、製造、防衛はどうつながっていますか？」
3. 「中央の装置を回した理由は何ですか？」
4. 「2ウェーブ目から増えたルールは何でしたか？」
5. 「危険な状態を画面のどこで判断しましたか？」
6. 「もう一度、自分から遊びたいですか？ そう思う／思わない理由は何ですか？」

回答は要約せず、本人の語句を短く記録する。ただし回答内容をAnalyticsへ送らず、テスト記録だけで管理する。

## 合格条件

最初の10人で、すべてを満たした時だけ次段階へ進む。

- 8/10以上が、誘導なしで採掘→製造→防衛を説明できる。
- 8/10以上が、最初の撃破を25秒以内に達成する。
- 7/10以上が、能動時間90秒へ到達する。
- 5/10以上が、促されずに2ラン目を開始する。
- 8/10以上が、W2で追加された周波数／共鳴の意味を概ね説明できる。
- 重要操作を妨げる文字切れ、重なり、誤タップが320×568または390×844で再現しない。
- クラッシュ、進行不能、復帰破損、バックグラウンド進行が0件。

一項目でも未達ならNO-GOとし、最も早い離脱点を一つ直して、新しい未経験者10人で再試験する。同じ人の学習後成績を新規理解率へ混ぜない。

## 次段階

10人ゲート通過後、30〜50人で次を確認する。

- 任意再挑戦率
- 平均セッション時間と中央値
- 1分到達率
- 3ビルドの選択分布と勝率
- 敗北後の再挑戦
- 端末・画面寸法別の離脱

この段階を通るまで、12ウェーブ化、敵6種化、強化18種化、開始プロトコル、広告、CrazyGames Basic Launch申請へ進まない。
